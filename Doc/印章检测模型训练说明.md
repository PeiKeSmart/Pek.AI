# 印章检测模型训练说明

## 结论

可以做，但不能凭空生成一个可用模型。

要得到可用于业务的印章检测模型，至少需要：

1. 一批真实业务样本
2. 对样本里的印章做框标注
3. 训练一个目标检测模型
4. 导出为 ONNX
5. 用当前 Demo 回归验证

当前仓库已经提供了 C# 推理 Demo，训练环节建议使用 YOLO。

## 建议的类别设计

如果你的目标只是统计总数，最简单是单类别：

- seal

如果后面还要区分类型，可以拆成：

- official_seal
- finance_seal
- contract_seal
- joint_seal
- e_signature_seal

一开始不要把类别拆得太细，否则样本量不足时精度会明显下降。

## 样本要求

建议至少准备下面这些场景：

1. 红色圆章
2. 红色椭圆章
3. 骑缝章
4. 黑白扫描件中的浅色章
5. 模糊、倾斜、压缩过的扫描件
6. 带红字、红表格、红 LOGO 的干扰页
7. 多章同页
8. 半个章、残缺章

经验建议：

- 快速验证：每类 100 到 300 张
- 可用模型：总样本 1000 张以上
- 业务稳定：总样本 3000 张以上

## 标注规范

使用矩形框框住完整印章区域。

标注时注意：

1. 同一页多个印章要分别框出
2. 骑缝章按你的业务口径决定是否分别标注
3. 不要把红色文字、红色表格线标成印章
4. 残缺章如果业务上需要计数，也要纳入标注

推荐工具：

1. labelImg
2. CVAT
3. Roboflow

## 数据集目录建议

建议整理为 YOLO 标准目录：

```text
dataset/
  images/
    train/
    val/
  labels/
    train/
    val/
```

标签文件采用 YOLO 格式：

```text
class_id center_x center_y width height
```

坐标全部是相对比例值，范围 0 到 1。

## 训练方式

仓库中已经提供模板文件：

- `Pek.AI.StampDetectionDemo/Training/stamp-seal.data.yaml`
- `Pek.AI.StampDetectionDemo/Training/train_yolo.py`

基本流程：

1. 安装 Python 3.10+
2. 安装 ultralytics
3. 修改数据集路径
4. 执行训练脚本
5. 导出 ONNX

安装命令：

```powershell
pip install ultralytics
```

训练命令示例：

```powershell
python .\Pek.AI.StampDetectionDemo\Training\train_yolo.py --data D:\dataset\stamp-seal.data.yaml --model yolov8n.pt --imgsz 640 --epochs 100
```

导出 ONNX 示例：

```powershell
yolo export model=runs\detect\train\weights\best.pt format=onnx imgsz=640 opset=12 simplify=True
```

## 接入当前 C# Demo

导出 ONNX 后，直接这样测试：

```powershell
dotnet run --project .\Pek.AI.StampDetectionDemo\Pek.AI.StampDetectionDemo.csproj -- --input D:\sample\seal.pdf --model D:\model\best.onnx --labels seal
```

如果训练时是多类别，需要把 `--labels` 按训练顺序传入，例如：

```powershell
dotnet run --project .\Pek.AI.StampDetectionDemo\Pek.AI.StampDetectionDemo.csproj -- --input D:\sample\seal.pdf --model D:\model\best.onnx --labels official_seal,finance_seal,contract_seal,joint_seal
```

## 训练建议

推荐从 `yolov8n` 或 `yolo11n` 这类轻量模型开始：

1. 训练快
2. 导出 ONNX 容易
3. C# 部署成本低

等你确认业务可行后，再考虑更大的模型。

## 常见问题

### 1. 能不能直接给一个通用印章模型？

理论上可以做一个通用版，但泛化效果高度依赖样本。

如果没有你业务里的真实 PDF、扫描件、电子章样本，做出来的模型通常只能演示，不能稳定上线。

### 2. OpenCV 已经能识别，为什么还需要模型？

OpenCV 更适合红色明显、背景简单的印章。

一旦遇到：

1. 黑白扫描
2. 红字干扰
3. 电子签章
4. 浅色章
5. 残缺骑缝章

误检和漏检都会明显增加。

### 3. 要多少数据才够？

如果只是做 Demo，几百张就能开始。

如果要业务可用，建议至少准备 1000 张以上的真实文档页，并保证干扰场景足够多。

## 下一步建议

你最应该先做的是两件事：

1. 先收集一批真实 PDF/图片样本
2. 明确“什么算一个印章”的业务口径

只要你把样本准备好，这个 Demo 已经可以直接作为 ONNX 推理端来验证模型效果。
